Архітектура:
Для кастмізації вибрано використовувати кастомний тотал як нативний шлях для вирішення такої задачі в М2. Тотал вибрано так-як зміни залежать від статуса користувача і це є рівень корзини, тому використано тотал а не плагін до ціни. 
Тотал викликає LoyaltyService для отримання розміру знижки. 
Спочатку перевіряєтсья чи є знижка в кеші, якщо ні йде виклик LoyaltyApiClient для отримання данних з зовнішнього серверу. 
Для тестового урл використовується метод getMockLoyaltyData що повертає тестові данні сумісні з реальним форматом. Данні записуються в кеш. 
Використовуєтсья CacheInterface і М2 відповідно до налаштувань магазину зберігає данні в редіс чи альтернативному кеші. Для відображення в корзині/чекауті додано кастомізацію леятуів(frone-end частина по суті).
